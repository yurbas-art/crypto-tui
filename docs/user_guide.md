# Руководство пользователя

## crypto-tui — приложение для шифрования и криптоанализа

TUI-приложение для работы с классическими шифрами: Цезарь, Виженер, Полибий,
а также для частотного анализа и взлома Виженера по индексу совпадений.

---

## Запуск

### Локально

```bash
git clone https://github.com/yurbas-art/crypto-tui.git
cd crypto-tui
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

### Через Docker

```bash
git clone https://github.com/yurbas-art/crypto-tui.git
cd crypto-tui
docker build -t crypto-tui .
docker run -it crypto-tui
```

## Управление

Приложение полностью управляется как мышью, так и клавиатурой.

| Действие | Клавиша |
|---|---|
| Переход между полями и кнопками | `Tab` / `Shift+Tab` |
| Нажать выделенную кнопку | `Enter` или `Space` |
| Выход из приложения | `q` |
| Палитра команд Textual | `Ctrl+P` |

> **Важно.** Если приложение запущено внутри терминального
> мультиплексора (`zellij`, `tmux`), мышиные клики по кнопкам
> могут не доходить до приложения из-за перехвата событий
> мультиплексором. В этом случае используйте навигацию
> клавиатурой (`Tab` + `Enter`) — она работает всегда.

## Главное меню

После запуска отображается меню с шестью пунктами:

```
1. Шифр Цезаря
2. Шифр Виженера
3. Шифр Полибия
4. Частотный анализ
5. Взлом Виженера (IC)
6. Выход
```

Выберите пункт клавишами `Tab` + `Enter` или кликом мыши.
На каждом экране есть кнопка **«← Назад»** для возврата в меню.

---

## 1. Шифр Цезаря

1. Введите текст в поле **«Текст»** (или загрузите из файла — см. ниже)
2. Введите числовой ключ в поле **«Ключ (число)»** (например, `3`)
3. Нажмите **«Зашифровать»** или **«Расшифровать»**
4. Результат появится в поле **«Результат»**

Поддерживаются отрицательные ключи и ключи больше 32 (вычисляются по модулю
размера алфавита). Символы вне русского алфавита (пробелы, пунктуация,
цифры) не изменяются.

---

## 2. Шифр Виженера

1. Введите текст
2. Введите ключевое слово в поле **«Ключевое слово»** (например, `КЛЮЧ`)
3. Нажмите **«Зашифровать»** или **«Расшифровать»**

Ключ автоматически приводится к верхнему регистру и циклически
повторяется до длины текста.

---

## 3. Шифр Полибия

Таблица 6×6 отображается прямо на экране:

```
    1  2  3  4  5  6
1   А  Б  В  Г  Д  Е
2   Ж  З  И  Й  К  Л
3   М  Н  О  П  Р  С
4   Т  У  Ф  Х  Ц  Ч
5   Ш  Щ  Ъ  Ы  Ь  Э
6   Ю  Я  .  ,  !  ?
```

- Для **шифрования** введите обычный текст
- Для **дешифрования** введите пары координат через пробел,
  например: `13 21 23 12 14 11`

Результат шифрования — строка пар координат, разделённых пробелом.

---

## 4. Частотный анализ (взлом шифра Цезаря)

1. Вставьте зашифрованный шифром Цезаря текст
2. Нажмите **«Анализировать»**

Программа покажет:
- вероятный ключ
- топ-10 самых частых букв текста с гистограммой
- автоматически расшифрованный текст

Метод работает надёжно на текстах от ~100 символов — чем длиннее текст,
тем точнее определяется ключ.

---

## 5. Взлом Виженера (индекс совпадений)

1. Вставьте зашифрованный шифром Виженера текст
2. Нажмите **«Взломать»**

Программа покажет:
- топ-5 вероятных длин ключа со значением индекса совпадений (IC)
- наиболее вероятную длину и сам ключ
- расшифрованный текст

**Ограничение метода.** Для надёжного определения каждой буквы ключа
требуется примерно 150–200 символов зашифрованного текста на одну
букву ключа. Например, для ключа длиной 4 нужен текст не короче
800–1000 символов. На более коротких текстах длина ключа обычно
определяется верно, но отдельные буквы ключа могут быть подобраны
с ошибкой — это математическое свойство частотного анализа на малой
выборке, а не ошибка программы.

## Работа с файлами

На каждом экране есть строка с полем пути к файлу и двумя кнопками:

```
Файл:
[ путь к файлу              ] [Открыть] [Сохранить]
```

**Открыть** — читает содержимое файла и подставляет его в поле «Текст».

**Сохранить** — записывает содержимое поля «Результат» в указанный файл.

### Требования к пути

- Путь указывается относительно директории, из которой запущено
  приложение (`python main.py`), либо абсолютный
- Поддерживаются только файлы с расширением `.txt`
- Файл должен быть в кодировке UTF-8

### Тестовые файлы проекта

В каталоге `test_data/` есть готовые файлы для проверки:

| Файл | Назначение |
|---|---|
| `plain_short.txt` | короткий исходный текст |
| `plain_long.txt` | длинный исходный текст (для частотного анализа) |
| `encrypted_caesar.txt` | пример для экрана «Шифр Цезаря» / «Частотный анализ» |
| `encrypted_vigenere.txt` | пример для экрана «Шифр Виженера» / «Взлом Виженера» |
| `encrypted_polybius.txt` | пример для экрана «Шифр Полибия» |
| `invalid.txt` | пустой файл — для проверки обработки ошибок |

Пример: чтобы продемонстрировать взлом Виженера, на экране
«Взлом Виженера (IC)» введите путь `test_data/encrypted_vigenere.txt`
и нажмите «Открыть», затем «Взломать».

## Возможные ошибки и их значение

| Сообщение | Причина |
|---|---|
| `[!] Введите текст` | поле текста пустое |
| `[!] Введите ключ` | поле ключа пустое (Цезарь/Виженер) |
| `[!] Ключ — целое число` | в поле ключа Цезаря введено не число |
| `[!] Укажите путь к файлу` | поле пути пустое при нажатии «Открыть»/«Сохранить» |
| `[!] Файл не найден: ...` | указанный путь не существует |
| `[!] Недопустимое расширение: ...` | файл не `.txt` |
| `[!] Файл пуст` | при чтении файл оказался пустым |
| `[!] Нет результата для сохранения` | поле «Результат» пустое при попытке сохранить |

## Запуск тестов

```bash
pip install -r requirements.txt
pytest tests/ -v
```

Проект содержит 29 тестов: unit-тесты для каждого алгоритма
(`tests/unit/`) и интеграционные тесты полного цикла
файл → шифр → результат (`tests/integration/`).
