# Архитектура приложения (исправленная версия)

## Общая схема архитектуры

Приложение построено по **трёхслойной архитектуре** с чётким разделением ответственности. Алгоритмический слой реализован в **модульном стиле** (функции без классов) — это идиоматичный подход для Python для чистых вычислительных алгоритмов.

```

│                    СЛОЙ ПРЕДСТАВЛЕНИЯ                    │
│                      (ui/textual_app.py)                 │
│  ┌──────────────────────────────────────────────────┐  │
│  │  TUI-интерфейс на базе Textual                   │  │
```

## Детальное описание слоёв

### 1. Слой представления (Presentation Layer)

**Модуль:** `ui/textual_app.py`

**Ответственность:**
- Отображение пользовательского интерфейса
- Приём ввода от пользователя
- Форматирование вывода данных
- Обработка пользовательских событий (нажатия клавиш, выбор меню)

**Технологии:**
- Библиотека **Textual** — современный фреймворк для создания TUI-приложений на Python
- Асинхронная модель работы (async/await)

**Компоненты интерфейса:**
- `MainMenu` — главное меню с выбором операций
- `TextInputScreen` — экран ввода текста или пути к файлу
- `KeyInputScreen` — экран ввода ключа шифрования
- `ResultScreen` — экран отображения результата
- `SaveFileDialog` — диалог сохранения результата

**Принципы:**
- Полное отсутствие бизнес-логики
- Все действия делегируются слою бизнес-логики
- Использование паттерна **MVC** (Model-View-Controller), где View — это Textual-виджеты

### 2. Слой бизнес-логики (Business Logic Layer)

#### Модуль: `services/cipher_service.py`

**Ответственность:**
- Координация работы с различными шифрами
- Валидация ключей и входных данных
- Управление текущим состоянием (открытый текст, зашифрованный текст)
- Предоставление унифицированного интерфейса для UI

**Ключевые методы:**
```python
class CipherService:
    def encrypt_caesar(self, text: str, key: int) -> str
    def decrypt_caesar(self, text: str, key: int) -> str
    def encrypt_vigenere(self, text: str, key: str) -> str
    def decrypt_vigenere(self, text: str, key: str) -> str
    def encrypt_polybius(self, text: str) -> str
    def decrypt_polybius(self, text: str) -> str
    def frequency_analysis(self, text: str) -> dict
    def auto_decrypt_caesar(self, text: str) -> tuple[str, int]
```

**Паттерн:** **Facade** — предоставляет упрощённый интерфейс к подсистеме шифров. Внутри методов происходит делегирование к модульным функциям:
```python
def encrypt_caesar(self, text: str, key: int) -> str:
    return caesar.encrypt(text, key)
```

---

#### Модуль: `services/file_service.py`

**Ответственность:**
- Чтение текстовых файлов
- Запись результатов в файлы
- Обработка ошибок ввода-вывода
- Валидация путей и расширений файлов
- Работа с кодировками (UTF-8 по умолчанию)

**Ключевые методы:**
```python
class FileService:
    def read_file(self, file_path: str) -> str
    def write_file(self, file_path: str, content: str) -> bool
    def validate_file_path(self, file_path: str) -> bool
    def get_file_info(self, file_path: str) -> dict
```

**Обработка ошибок:**
- `FileNotFoundError` — файл не найден
- `PermissionError` — нет прав доступа
- `UnicodeDecodeError` — неверная кодировка
- `IsADirectoryError` — указан путь к директории

### 3. Алгоритмический слой (Algorithm Layer)

Каждый модуль реализует **один конкретный алгоритм** через набор функций и следует принципу **Single Responsibility**. Используется **модульный стиль** (функции без классов) — это идиоматичный подход для Python для чистых вычислительных алгоритмов.

#### Модуль: `ciphers/caesar.py`

**Ответственность:** Реализация шифра Цезаря

**Константы модуля:**
```python
ALPHABET = "АБВГДЕЖЗИЙКЛМНОПРСТУФХЦЧШЩЪЫЬЭЮЯ"  # 32 буквы
ALPHABET_SIZE = len(ALPHABET)
```

**Публичные функции:**
```python
def encrypt(text: str, key: int) -> str
def decrypt(text: str, key: int) -> str
```

**Математическая основа:**
- Шифрование: `E(x) = (x + k) mod n`
- Дешифрование: `D(x) = (x - k) mod n`

**Особенности:**
- Сохранение регистра символов
- Игнорирование символов, не входящих в алфавит (пробелы, знаки препинания)
- Поддержка отрицательных ключей
- Дешифрование реализовано как шифрование с отрицательным ключом: `decrypt(text, key) = encrypt(text, -key)`

---

#### Модуль: `ciphers/vigenere.py`

**Ответственность:** Реализация шифра Виженера

**Константы модуля:**
```python
ALPHABET = "АБВГДЕЖЗИЙКЛМНОПРСТУФХЦЧШЩЪЫЬЭЮЯ"
ALPHABET_SIZE = len(ALPHABET)
```

**Публичные функции:**
```python
def encrypt(text: str, key: str) -> str
def decrypt(text: str, key: str) -> str
```

**Математическая основа:**
- Шифрование: `E(xᵢ) = (xᵢ + kᵢ) mod n`, где `kᵢ` — i-й символ ключа
- Дешифрование: `D(xᵢ) = (xᵢ - kᵢ) mod n`

**Особенности:**
- Автоматическое циклическое расширение ключа до длины текста
- Приведение ключа к верхнему регистру
- Сохранение регистра исходного текста
- Символы не из алфавита пропускаются, но индекс ключа не сдвигается

---

#### Модуль: `ciphers/polybius.py`

**Ответственность:** Реализация шифра Полибия (квадрат Полибия)

**Константы и глобальные структуры модуля:**
```python
ALPHABET = "АБВГДЕЖЗИЙКЛМНОПРСТУФХЦЧШЩЪЫЬЭЮЯ"  # 32 буквы
TABLE_SIZE = 6  # фиксированный размер таблицы

# Глобальные структуры данных (заполняются при импорте):
TABLE: list[list[str]] = []                    # таблица 6×6
CHAR_TO_COORDS: dict[str, tuple[int, int]] = {}  # символ → (строка, столбец)
COORDS_TO_CHAR: dict[tuple[int, int], str] = {}  # (строка, столбец) → символ
```

**Публичные функции:**
```python
def build_table() -> None        # инициализация таблицы (вызывается при импорте)
def encrypt(text: str) -> str    # текст → строка пар цифр
def decrypt(text: str) -> str    # строка пар цифр → текст
```

**Структура таблицы (фиксированная 6×6):**
```
    1  2  3  4  5  6
1:  А  Б  В  Г  Д  Е
2:  Ж  З  И  Й  К  Л
3:  М  Н  О  П  Р  С
4:  Т  У  Ф  Х  Ц  Ч
5:  Ш  Щ  Ъ  Ы  Ь  Э
6:  Ю  Я  .  ,  !  ?
```

**Особенности:**
- Таблица фиксированного размера 6×6 = 36 ячеек
- 32 буквы русского алфавита + 4 знака препинания (`.`, `,`, `!`, `?`)
- Координаты нумеруются с 1
- Результат шифрования — строка пар цифр, разделённых пробелами
- Функция `build_table()` вызывается один раз при импорте модуля

#### Модуль: `analysis/frequency.py`

**Ответственность:** Частотный анализ текста

**Константы модуля:**
```python
ALPHABET = "АБВГДЕЖЗИЙКЛМНОПРСТУФХЦЧШЩЪЫЬЭЮЯ"
ALPHABET_SIZE = len(ALPHABET)

# Эталонные частоты букв русского языка
RU_FREQ: dict[str, float] = {
    "О": 0.1097, "Е": 0.0845, "А": 0.0801, "И": 0.0735, "Н": 0.0670,
    "Т": 0.0626, "С": 0.0547, "Р": 0.0473, "В": 0.0454, "Л": 0.0440,
    "К": 0.0349, "М": 0.0321, "Д": 0.0298, "П": 0.0281, "У": 0.0262,
    "Я": 0.0201, "Ы": 0.0190, "Ь": 0.0174, "Г": 0.0170, "З": 0.0165,
    "Б": 0.0159, "Ч": 0.0144, "Й": 0.0121, "Х": 0.0097, "Ж": 0.0094,
    "Ш": 0.0073, "Ю": 0.0064, "Ц": 0.0048, "Щ": 0.0036, "Э": 0.0032,
    "Ф": 0.0026, "Ъ": 0.0004,
}
```

**Публичные функции:**
```python
def count_frequencies(text: str) -> dict[str, int]
def relative_frequencies(text: str) -> dict[str, float]
def guess_caesar_key(text: str) -> int
def auto_decrypt_caesar(text: str) -> tuple[str, int]
```

**Алгоритм определения ключа:**
1. Подсчёт абсолютных частот каждого символа в тексте
2. Вычисление относительных частот: `p = nᵢ / N`
3. Поиск наиболее частого символа в шифртексте
4. Сопоставление с наиболее частым символом русского языка (`О`)
5. Вычисление сдвига: `key = (encrypted_index - expected_index) mod n`

**Особенности:**
- Символы не из алфавита игнорируются
- Для пустого текста возвращается ключ 0
- `auto_decrypt_caesar` импортирует `decrypt` из `caesar.py` и возвращает кортеж `(расшифрованный_текст, ключ)`

## Паттерны проектирования

### 1. **Facade** (Фасад)

**Где используется:** `CipherService`

**Зачем:** Предоставляет упрощённый интерфейс к подсистеме модульных функций шифров. UI не должен знать о существовании модулей `caesar`, `vigenere`, `polybius`.

```python
# UI работает только с CipherService
service = CipherService()
result = service.encrypt_caesar(text, key)

# Внутри CipherService происходит делегирование:
import caesar
def encrypt_caesar(self, text, key):
    return caesar.encrypt(text, key)
```

---

### 2. **MVC** (Model-View-Controller)

**Где используется:** Архитектура UI

- **Model** — данные (текст, ключи, результаты)
- **View** — Textual-виджеты (отображение)
- **Controller** — обработчики событий в `textual_app.py`

---

### 3. **Singleton** (Одиночка) — планируемый

**Где планируется:** `FileService`

**Зачем:** Гарантирует, что в приложении существует только один экземпляр сервиса для работы с файлами.

## Расширение и модификация

### Добавление нового шифра

1. Создать новый модуль в `ciphers/` (например, `playfair.py`)
2. Реализовать функции `encrypt()` и `decrypt()` по аналогии с существующими
3. Добавить методы в `CipherService`:
   ```python
   def encrypt_playfair(self, text: str, key: str) -> str:
       import playfair
       return playfair.encrypt(text, key)
   ```
4. Добавить пункт меню в UI

**Время разработки:** ~30 минут

---

### Добавление нового метода анализа

1. Создать новый модуль в `analysis/` (например, `index_of_coincidence.py`)
2. Реализовать функции анализа
3. Добавить метод в `CipherService`
4. Добавить пункт меню в UI

## Преимущества выбранной архитектуры

1. **Модульность** — каждый компонент можно разрабатывать и тестировать независимо
2. **Идиоматичность для Python** — модульные функции для чистых алгоритмов проще и понятнее классов
3. **Расширяемость** — добавление новых шифров не требует изменения существующего кода
4. **Тестируемость** — каждый слой можно покрыть unit-тестами
5. **Разделение ответственности** — UI не знает о деталях реализации шифров
6. **Повторное использование** — алгоритмы можно использовать вне TUI (например, в веб-интерфейсе)
7. **Читаемость кода** — чёткая структура облегчает понимание и поддержку

## Ограничения и возможные улучшения

### Текущие ограничения:
- Поддержка только русского алфавита (32 буквы без `Ё`)
- Отсутствие поддержки многопоточности
- Нет кеширования результатов анализа
- Модульные функции не объединены общим интерфейсом

### Возможные улучшения:

1. **Введение абстрактного базового класса `CipherStrategy`:**
   ```python
   from abc import ABC, abstractmethod
   
   class CipherStrategy(ABC):
       @abstractmethod
       def encrypt(self, text: str, key: Any) -> str: ...
       @abstractmethod
       def decrypt(self, text: str, key: Any) -> str: ...
   ```
   Это позволит использовать паттерн **Strategy** и упростит добавление новых шифров.

2. **Поддержка буквы `Ё`** — расширение алфавита до 33 символов.

3. **Поддержка других алфавитов** (английский, смешанные тексты).

4. **Асинхронная обработка больших файлов** (использование `asyncio`).

5. **Графический интерфейс** (PyQt/Tkinter) как альтернатива TUI.

6. **Веб-интерфейс** (Flask/FastAPI).

7. **Экспорт результатов** в различные форматы (JSON, CSV, PDF).

8. **База данных** для хранения истории шифрований.
